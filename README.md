## Overview

Commerce Catalyst gives Adobe Commerce teams a structured way to bring AI into day-to-day commerce operations. Instead of bolting features on one by one, it centers on orchestration, integration, and automation so commerce experiences can be assembled around data signals, business logic, and app builder workflows.

The SDK is a strong fit for projects that need to coordinate several AI providers, support deployment across different environments, and combine commerce intelligence capabilities into a single implementation path. That makes it easier to build tools that react to inventory changes, pricing context, and customer behavior without manually stitching every layer together.

## How to Use It

Commerce Catalyst is meant to act as an orchestration layer for AI tasks tied to commerce operations. A common flow looks like this:

1. Configure your Adobe Commerce connection.
2. Select the AI provider or providers you want to use.
3. Define the automation or orchestration rule you want to run.
4. Wire the SDK into your app builder or deployment target.
5. Monitor inventory, pricing, and customer journey outputs as part of your commerce flow.

Example workflow pattern:

- Connect commerce events to an orchestration pipeline
- Route events to the appropriate AI service
- Return pricing, inventory, or analytics signals to the storefront or backend
- Extend the integration across platforms where needed
